แก้ปัญหาข้อมูลจากหน้าแอดมินไม่แสดงบนหน้าเว็บลูกค้า

รายการที่แก้
1. หน้าแรกดึงเมนูแนะนำจากฐานข้อมูลจริง (featured_menu) แทนข้อมูลตัวอย่างที่เขียนค้างไว้
2. หน้าเมนูและแกลเลอรีใช้ URL อัปโหลดที่ถูกต้องผ่าน Flask routes
3. ข้อมูลชื่อร้าน เบอร์โทร อีเมล ที่อยู่ และ Social Media ใน Footer ดึงจากหน้า Admin Settings
4. ปิด HTML cache เพื่อให้ข้อมูลที่แก้ไขแสดงทันทีหลังบันทึก/รีเฟรช
5. เรียก initialize_database ตอน Gunicorn เริ่มทำงานด้วย

หมายเหตุ Render
- SQLite และไฟล์ที่อัปโหลดไว้ในโฟลเดอร์ uploads บน Render อาจหายเมื่อบริการ restart/redeploy เพราะ filesystem ไม่ถาวร
- สำหรับใช้งานจริงควรใช้ Render PostgreSQL และ Cloudinary/S3 สำหรับรูปภาพ
